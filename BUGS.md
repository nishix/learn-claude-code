# 🐛 仕込まれた不具合リスト

このドキュメントは、ハンズオン勉強会用に意図的に仕込まれた不具合の一覧です。
Claude Code を使って、これらの不具合を見つけ出し、修正してください！

---

## 🔴 初級：UI/表示バグ

### バグ1: 統計表示が更新されない

**症状**
- Todoを追加・削除・完了にしても、統計表示が「全体: 0 | 完了: 0」のままになっている

**発生箇所**
- ファイル: `src/App.jsx`
- 画面上部の統計表示エリア

**ヒント**
- React の状態管理（useState）に関連
- 統計情報が別の状態で管理されているか確認してみてください

---

### バグ2: 削除ボタンが表示されない

**症状**
- Todoリストの各アイテムに削除ボタンが表示されていない
- ボタンの要素は DOM に存在するが、画面上に見えない

**発生箇所**
- ファイル: `src/App.css`
- `todo-item button` のスタイル

**ヒント**
- CSS の display プロパティを確認してみてください

---

## 🟠 中級：ロジックバグ

### バグ3: 空白のみのTodoが追加できる

**症状**
- スペースやタブだけのテキストを入力して「追加」を押すと、空白のTodoがリストに追加される
- 本来は空白のみの入力は無視すべき

**発生箇所**
- ファイル: `src/App.jsx`
- `addTodo()` 関数の条件分岐

**ヒント**
- `input` と `input.trim()` の違いを考えてみてください
- `.trim()` メソッドは何をしているのか確認してみては？

---

### バグ4: チェックボックスの状態が反映されない

**症状**
- Todoのチェックボックスをクリックして完了にしようとしても、チェックが入らない
- テストも失敗している

**発生箇所**
- ファイル: `src/App.jsx`
- `todo-item` 内の input[type="checkbox"]

**ヒント**
- チェックボックスの `checked` 属性の値を確認してみてください
- 常に特定の値に固定されていないか？

---

## 💡 学習のコツ

1. **エラーメッセージを読む** - テスト結果のエラーメッセージに、多くの情報が含まれています
2. **Claude Code を活用** - ファイルを検索したり、コードを読んだり、修正を提案してもらったりしてみましょう
3. **テストを実行する** - 修正後は `npm test` で本当に直ったか確認してください
4. **段階的に進める** - 1つの不具合から始めて、順番に修正していくのがおすすめです

---

## 🚀 修正の進め方（例）

1. バグ2（削除ボタンが表示されない）から始める → CSSが簡単
2. バグ1（統計表示が更新されない）を修正 → React の状態管理を学べる
3. バグ3（空白のみのTodoが追加できる）を修正 → JavaScript の文字列操作
4. バグ4（チェックボックスの状態が反映されない）を修正 → React の条件付きレンダリング

Happy debugging! 🎉
